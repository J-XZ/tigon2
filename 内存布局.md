# TigonKV 共享布局

TigonKV 使用同一个 ivshmem backing 文件映射到各 VM。物理介质仍是宿主机普通
DRAM；`HWCC`、`SWCC` 是协议和实验统计中的逻辑类别，不是真实 CXL DIMM。

文件开头是 `SharedLayoutHeader`（实现中的 `SharedHeader`），随后是持久化
open-addressed `uint32_t` shared index，再随后是固定大小的 offset 可定位 slot。
index、layout 和 shared row metadata 按逻辑 HWCC 记账；slot 的 key/value 都在映射
内，进程本地只保存文件描述符、映射地址和统计描述符，不保存按 KV 数量增长的
DRAM shadow copy。index publication 在 slot 完整初始化并完成所需 write-back 后进行。

每个 slot 的状态为 `EMPTY -> PRIVATE | SHARED_ACTIVE -> TOMBSTONE`。private slot
由 `owner = stable_hash(key) % vm_count` 的 VM 访问；promotion 后 slot 的 value
成为 shared SWCC payload，逻辑 HWCC 只记录行状态、owner、版本和生命周期。

当前布局采用 global allocator 语义：同一个 backing 中的地址可能交错，缓存一致性
类别始终由显式状态/domain 选择，不能从地址猜测。配置保留 HWCC/SWCC offset 和
容量，以便和 cxlkv 的实验配置逐字段对齐；这不是对宿主机物理页的区域承诺。

持久引用只使用 `{domain, offset}`，不写 raw pointer。clean checkpoint 在计时窗口
之外执行 `msync` 和 epoch 标记；active/dirty backing 不被静默当成 clean restart。
