# TigonKV 双区域内存布局

TigonKV 的共享 backing 由 `DualRegionMappedPool` 打开，并按配置中 `hwcc` 与
`swcc` 的 offset/size 划分为两个不可重叠的持久区域。它是 NUMA host-DRAM/ivshmem
模拟的布局，不是对真实 CXL 物理介质的声明。

```text
backing mapping
├─ HWCC [hwcc.offset_mb, hwcc.offset_mb + hwcc.size_mb)
│  ├─ SharedLayoutHeader、partition 目录和 allocator shard 元数据
│  ├─ 每 partition 的 shared B+Tree 节点
│  ├─ shared-row metadata、transport ring 和 EBR 元数据
│  └─ shared index 根与统计
└─ SWCC [swcc.offset_mb, swcc.offset_mb + swcc.size_mb)
   ├─ owner-private B+Tree 节点和 PrivateRow
   └─ 已迁移行的 shared payload
```

所有持久引用使用 `RegionOffset`，而非进程虚拟地址；独立进程或 VM 在不同 mmap
基址 attach 后仍能解析同一对象。初始化和 attach 会校验 layout header、配置 hash、
VM/partition 数、固定 key/value 尺寸及区域边界。分配器按 `PoolKind` 记账，跨区域
释放和越界 offset 都是硬错误。

未迁移数据仅存在 owner 的 SWCC 私有行。首次非 owner 访问时，引擎在 SWCC 分配
shared payload，发布 shared metadata/index 后将该行视为 migrated；owner 之后也从
shared payload 读写。move-out 在没有读者、索引已摘除且 EBR 回收条件满足后回收
shared 对象，避免私有壳和共享 payload 双权威。

实现和回归证据位于 `kv/engine/region_allocator.*`、`kv/engine/kv_types_layout.h`、
`tests/region_allocator_test.cpp` 与 `tests/kv_partition_test.cpp`。真实多 VM attach
与 NUMA 落位仍须在获授权的 VM 拓扑上验收。
