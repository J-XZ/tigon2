# TigonKV 延迟插入审计

当前实现以 `kv/engine/mem_access.h` 作为唯一的引擎侧访问记录入口。全局
`LatencySimulator` 由 `KVStore` 按 `latency_inject` 配置，门面每个前台操作在
安全点建立/结束作用域；延迟只在该作用域退出后补齐。

已覆盖：

- `KVPartition` 私有行 value 的读写；
- `BTreeOLC_CXL` 的 lookup/insert/remove/scan 入口：按实例绑定的 private arena
  或 shared index 域分别记为 SWCC/HWCC；
- `MPSCRingBuffer` 的固定消息发送和接收；
- `SCCManager` 的 cacheline flush/writeback，以及 checkpoint 的 allocator
  flush 范围；
- `DumpStats` 的 HWCC/SWCC 原始访问、缓存命中/未命中及累计延迟字段。

尚待 M6 后续接线：共享 payload 读写。它必须继续通过此 wrapper 接入，不能在协议
代码中散落独立统计或忙等。
