# TigonKV 延迟插入审计

当前实现以 `kv/engine/mem_access.h` 和 cxlkv 移植的
`kv/engine/latency_inject.*` 作为唯一的引擎侧访问记录入口。全局
`latency_sim::LatencySimulator` 由 `KVStore` 按 `latency_inject` 全字段配置，门面
每个前台操作建立 `kForeground` scope；延迟只在最外层 scope 退出后补齐。

已覆盖：

- `KVPartition` 私有行 value 的读写；
- `BTreeOLC_CXL` 的 lookup/insert/remove/scan 入口：按实例绑定的 private arena
  或 shared index 域分别记为 SWCC/HWCC；
- `MPSCRingBuffer` 的固定消息发送和接收；
- `SCCManager` 的 cacheline flush/writeback，以及 checkpoint 的 allocator
  flush 范围；
- `KVPartition` 的 migrated shared payload：GET/PUT/CAS/INCR、move-in/out
  和 SCAN 的 SCC read/write；
- `DumpStats` 的 cxlkv 同名 `swcc_raw`/`hwcc_raw`、命中/未命中、命中率与分域
  delayed_ns 字段。

移植保留 TSC 校准、线程本地 generation 状态、组相联 LRU、xorshift fixed-hit、
`RecordRange`/`RecordLine`、`ScopeKind` 和 `SnapshotStats`/`TakeStatsAndReset` API；
关闭注入时 wrapper 只经过 relaxed fast gate。`latency_modes_test` 覆盖禁用、范围
折行、fixed-hit、LRU、命中禁用及统计 reset。
