# 对象访问分类与一致性设计

协议前提：多个 VM 通过 ivshmem 映射同一宿主机 DRAM。底层 CPU coherence 可能掩盖
SWCC 错误，因此 correctness 测试使用 `NonCoherentSwccTestBackend`；延迟模拟独立
于真实 write-back/invalidate。

|对象|AccessClass / domain|可访问者|权威副本与同步|普通 flush|发布/回收|
|---|---|---|---|---|---|
|transaction buffer|process-local / DRAM|当前进程|worker 栈/普通同步|否|操作结束|
|message buffer|process-local / DRAM|当前进程|transport 线程|否|发送完成|
|owner B+Tree page|owner-private / private SWCC|partition owner VM|进程内 mutex|否|checkpoint dirty range|
|owner key/value row|owner-private / private SWCC|owner VM|共享文件 mutex 下的本地 owner 检查|否|tombstone 后 slot 复用|
|owner local metadata|owner-private / private SWCC|owner VM|固定布局字段、进程内保护|否|clean attach 校验|
|owner allocator metadata|owner-private / private SWCC|owner VM|arena 状态|否|checkpoint|
|shared B+Tree/index|shared HWCC / hwcc-index|所有 VM|跨进程 mutex/原子发布|否|tombstone/EBR|
|shared row metadata|shared HWCC / hwcc-metadata|所有 VM|行状态、版本、ref count|否|retire 后 reclaim|
|shared row payload|shared SWCC / payload|所有 VM|shared row lock + SCC write-through|写者完成时|ref count 为零后 reclaim|
|migration tracker|owner-private 或 HWCC|owner；跨节点状态时所有 VM|状态机|否|状态终结后 reclaim|
|EBR metadata|shared HWCC / hwcc-ebr|所有 VM|epoch 与 ref 保护|否|shutdown drain|
|retired object|shared HWCC bookkeeping + 原 domain|EBR 参与者|不可达后 epoch reclaim|否|安全 epoch 回收|
|global layout/root|shared HWCC / hwcc-layout|所有 VM|release/acquire + mutex|否|clean epoch|
|transport buffer|process-local / transport domain|当前进程|消息队列|否|消息消费|
|checkpoint metadata|shared HWCC / layout|所有 VM|clean/active marker|checkpoint 时|下次 attach|
|trace/runtime stats|process-local / DRAM|当前进程|原子计数器|否|进程退出|

private PUT/GET 直接访问 owner-private SWCC，不创建 shared row metadata，也不调用
SCC。非 owner 首次访问在同一把共享锁下完成 promotion，先完整初始化 payload，再
发布 `SHARED_ACTIVE`；该状态写入是 remote lookup 的可达性线性化点。shared update
在共享锁内完整写 value，执行 write-through `msync` 作为实验中的可见性边界后释放锁。
reader 只在锁内复制完整 value。真实机器上的 `msync` 不能替代 NonCoherent 后端的
显式 write-back/invalidate 测试。

move-out 在本实现中通过 owner-only delete/reinsert 生命周期完成；默认不复用留下
shared index 的 payload。clean restart 只接受 clean epoch，crash recovery 不在本实验
承诺范围内。
