# TigonKV 缓存一致性设计

TigonKV 以 `SCCManager` 的 WriteThrough 路径维护已迁移 shared payload 的可见性。
它不把 `msync`、checkpoint 或宿主 CPU 的硬件一致性当作跨 VM 一致性协议；本项目的
实验结果只能描述为 NUMA-based CXL shared-memory emulation。

| 对象/路径 | 区域 | 一致性责任 |
|---|---|---|
| owner-private row、private B+Tree | SWCC | 仅 owner 访问，不调用 SCC |
| shared B+Tree/index、row metadata | HWCC | 原子元数据与 B+Tree OLC |
| migrated shared payload | SWCC | 行锁下 SCC `prepare_read`/`do_read`/`do_write`/`finish_write` |
| transport ring | HWCC | MPSC 的 release/acquire 发布与消费 |
| checkpoint 元数据 | HWCC | clean-attach/持久化辅助，不替代行协议 |

读取 shared payload 时，SCC 按 host readable bitmap 判断 read hit；miss 先执行必要的
flush/失效操作再读取。写路径在行写锁内更新 payload，`finish_write` 清除其他 host 的
readable 状态并完成 write-back，之后才释放该行。未迁移私有路径必须保持
`private_path_scc_calls == 0`；migrated 行（包括 owner 访问）不得回读私有壳。

`tests/scc_test_backend.*` 提供确定性的 NonCoherent 测试后端：各模拟 host 有独立
cache copy，只有正确的 flush/write-back 才会更新 visible copy。`scc_protocol_test`、
`kv_shared_protocol_test` 和 `kv_partition_test` 覆盖 read hit/miss、写回、迁移发布和
负向可见性场景。真实 ivshmem VM 的持续 e2e 验收仍未执行。
