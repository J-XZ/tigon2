本文件对应主仓库 `tigonkv-plan` 分支的归档提交。它保存当前可继续开发、复查
和重跑验证所需的源码、YCSB 子模块构建快照与 `exp_data/` 实验记录；不表示
PLAN.md 的所有验收项已经完成。

主仓库中被 `.gitignore` 排除的 `build-relwithdebinfo/`、VM 镜像缓存和 Python
缓存没有提交：它们是主机相关的派生文件，且可用下文命令从已提交的源码和配置重建。
这避免将约 500 MiB 的本机构建产物与不可移植 VM 临时镜像写入源码历史。

## 从干净主机恢复

```bash
git clone --recurse-submodules https://github.com/J-XZ/tigon2.git
cd tigon2
git checkout tigonkv-plan
git submodule update --init --recursive

cmake -S . -B build-relwithdebinfo -DCMAKE_BUILD_TYPE=RelWithDebInfo
cmake --build build-relwithdebinfo -j2
ctest --test-dir build-relwithdebinfo --output-on-failure
```

真实 VM 验证使用仓库中的 `experiment_config.jsonc` 与
`scripts/e2e/run_guest_e2e_workflows.sh`。先按该脚本的既有前提准备可访问的四台
VM、SSH 端口和共享内存布局，再从新的干净 pool 运行；不应依赖已回收服务器上的
VM 进程、guest 磁盘临时状态或共享内存内容。归档期间没有重挂 SSD，也没有调整
服务器网络配置。

复跑 mixed-only E 的命令模板如下（正式验收应使用独立日志目录并完成 3 轮）：

```bash
TIGONKV_E2E_BINARY_DIR="$PWD/build-relwithdebinfo" \
TIGONKV_POOL_INITER="$PWD/build-relwithdebinfo/cxl_pool_initer" \
TIGONKV_E2E_THREADS=4 TIGONKV_E2E09_MIXED_ONLY=1 \
TIGONKV_E2E_MIXED_SECONDS=60 TIGONKV_E2E_TIMEOUT_SEC=180 \
scripts/e2e/run_guest_e2e_workflows.sh \
  "$PWD/exp_data/supplemental-e-mixed/logs-3-clean" 3 09
```
