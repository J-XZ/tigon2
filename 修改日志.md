## 架构决策

* KV 对外只暴露统一 namespace；table/partition/owner 仅在内部路由。
* stable FNV-1a hash 计算 partition，owner 为 `partition % vm_count`。
* 固定布局 mmap slot 使用 offset 可重建引用；private slot 只有 owner 逻辑访问，
  remote access 在共享锁内 promotion 为 `SHARED_ACTIVE`。
* shared payload 使用 Tigon SCC WriteThrough 语义：完整写入、write-back、释放
  shared lock；本实现的文件-backed test path 用 `msync` 表示实际 write-through。
* 首次 shared publication 的线性化点是 slot state 从 PRIVATE 写为 SHARED_ACTIVE，
  并且在这之前初始化整个 value、key 和 metadata。
* `RETIRING` 的实验约束由 tombstone + 不可达 slot + reclaim counter 实现；默认
  不缓存无 shared index 的 payload。
* `NonCoherentSwccTestBackend` 显式区分每 host cache 和 visible backing，证明缺少
  write-back/invalidate 时的 stale read。
* 延迟模拟按显式 PoolKind/AccessKind 计数，不能替代 SCC correctness。

## allocator 决策

现有闭源静态库 SHA256：
`1048cfb372ddc5feea484a881cf4e8dfa153a5938f6ec83ec88343f0a5b39636`。
由于本机没有 `/dev/ivpci0`、backing 或 QEMU，原 cxlalloc 的真实 ivshmem multi-VM
capability 不能宣称通过。已实现并运行普通 mmap 文件的 capability matrix：offset
稳定、attach/restart、跨实例共享、process-shared lock、free/reuse、OOM、容量/域
统计、checkpoint 和重复 migration。最终选择 `allocator_mode=global`：一个共享
映射加显式逻辑 domain，`physical_region_split=0`，HWCC 逻辑预算固定不超过 1 GiB，
`unclassified_shared_bytes=0`。真实设备能力若失败，下一步应切换 dual-region，不能
silent fallback。

## 最终统计接口

`KVStore::DumpStats()` 输出 `TIGONKV_MEMORY_STATS`、`TIGONKV_RUNTIME_STATS`，包括
logical HWCC/SWCC usage、owner-private/shared payload、allocator overhead、unclassified
bytes、active rows、promotion、private/shared flush 和 reclaim。checkpoint flush 在
运行时窗口之外计数；普通 private operation 不计 SCC flush。

## 已知限制

1. 真实 cxlalloc ivshmem backend 尚未在缺失设备的环境中验证；当前 global wrapper 使用
   mmap backing，不能替代真实多 VM 页一致性测试。
2. 当前公共测试覆盖的是可重复本地/多实例模型；SSH runner 和 VM helper 不会在无拓扑
   时自动创建或重启资源。
3. crash recovery 不在承诺范围内；active/dirty 状态不得被当作 clean restart。
4. 当前环境无 QEMU、`/dev/ivpci0` 和正式 backing，因此多 VM SSH、真实页 NUMA
   placement、完整规模多轮 E2E/YCSB 尚未执行；安全脚本只做状态检查并 hard fail。
